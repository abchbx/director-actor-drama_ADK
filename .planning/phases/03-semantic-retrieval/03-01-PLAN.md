    b) **`_get_tag_weight(tag: str) -> float`**: Extract prefix from "前缀:值" tag, return TAG_WEIGHTS[prefix] or 1.0 for no-prefix.

    c) **`_compute_tag_score(query_tags: list[str], entry_tags: list[str]) -> float`**: Per D-10:
       - For each query_tag, check if it matches any entry_tag (exact match or value-only match where `qt.split(":")[-1] == et.split(":")[-1]`)
       - Exact match: score += max(weight(qt), weight(et))
       - Value-only match (no prefix on query): score += 1.0 (lower weight to avoid gaming, per RESEARCH.md Open Question 1 resolution)

    d) **`_normalize_scene_range(scenes_covered) -> set[int]`**: Convert "3-5" → {3,4,5}, integer 3 → {3}, handle edge cases. Per Pitfall 5 in RESEARCH.md.

    e) **`_search_scene_summaries(query_tags, summaries) -> list[dict]`**: Per D-06/D-08, iterate scene_summaries, compute _compute_tag_score for each, return results with: source="scene_summaries", scenes_covered, text=summary, matched_tags, score.

    f) **`_search_text_layer(query_tags, entries, source_name) -> list[dict]`**: Per D-11, for each entry check if any query tag value (after splitting ":") is contained in entry text. Fixed weight 1.0. Return: source=source_name, scenes_covered=str(scene), text=entry, matched_tags, score=1.0.

    g) **`_dedup_results(results: list[dict]) -> list[dict]`**: Per D-12, group by _normalize_scene_range(scenes_covered), keep highest score per scene set overlap. Two entries overlap if their normalized scene ranges intersect.

    h) **`retrieve_relevant_scenes(tags, current_scene, tool_context, actor_name=None, top_k=5) -> list[dict]`**: Per D-05/D-06/D-07/D-08:

       - If actor_name provided: search only `state["actors"][actor_name]`; else search all actors
       - Collect scene_summaries, working_memory, critical_memories
       - Run _search_scene_summaries on scene_summaries
       - Run _search_text_layer on working_memory (source_name="working_memory") and critical_memories (source_name="critical_memories")
       - Combine, _dedup_results, sort by score descending, return top_k
       - Each result dict: {"source": str, "scenes_covered": str, "text": str, "matched_tags": list[str], "score": float}

    i) **`_extract_auto_tags(actor_data, tool_context) -> list[str]`**: Per D-15, extract tags from:
       - Latest working_memory entry text (take key nouns/phrases as tags — simple heuristic: extract character names from actor_data knowledge about other characters, and key nouns from entry text)

       - Parse tags with _parse_tags_from_llm_output
       - Update scene_summaries entries with tags
       - Set `state["drama"]["tags_backfilled"] = True`
       - Return dict with count of tagged summaries

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM output → tag parsing | Untrusted LLM output crosses into structured tag data |
| User tag input → retrieval | Tags from Tool function input enter matching algorithm |
| backfill_tags → LLM call | Existing summaries sent to LLM for tag generation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | tag input validation | mitigate | Validate tag length ≤ 50 chars per tag, prefix whitelist check in TAG_WEIGHTS, reject tags containing special chars like `{}``<``>` |
| T-03-02 | Information Disclosure | retrieve_relevant_scenes actor_name filter | mitigate | Internal function enforces actor_name filtering — actor_name parameter cannot be overridden externally to access other actors' memories (D-07) |
| T-03-03 | Tampering | LLM output tag injection | accept | Tags are metadata for search only, not rendered in UI or used for control flow. Malformed tags simply won't match — low impact |
</threat_model>

       - 取 new_fact 前 20 字，与每个 existing fact 的 fact 字段前 20 字比较
       - 计算相同字符占比（交集/并集），> 80% 则标记重叠
       - 返回 `{"is_duplicate": bool, "overlapping_with": str | None}`

    4. **`_generate_fact_id(fact: str, current_scene: int, established_facts: list[dict]) -> str`**（per D-04）：
       - 使用 `re.search(r'[\u4e00-\u9fff]{2,4}', fact)` 提取关键词

       - 格式 `fact_{scene}_{keyword}_{index}`，index 为同前缀的已有数量+1

    5. **`add_fact_logic(fact: str, category: str, importance: str, state: dict) -> dict`**（per D-01/D-06/D-09/D-10/D-11）：
       - 验证 category 在 FACT_CATEGORIES 中，不在则返回 error
       - 验证 importance 为 "high"/"medium"/"low"，不在则返回 error
       - 调用 _check_fact_overlap，重叠时返回 `{"status": "info", "message": "⚠️ 可能重复：..."}`
       - 检查 MAX_FACTS，超限返回 info 提醒
       - 调用 _extract_actor_names 提取 actors
       - 调用 _generate_fact_id 生成 fact_id
       - 创建事实对象 `{"id", "fact", "category", "actors", "scene", "importance", "added_at"}`
       - 追加到 state["established_facts"]
       - 返回 `{"status": "success", "fact_id": ..., "fact": new_fact}`

    6. **`_filter_relevant_facts(state: dict) -> list[dict]`**（per D-16）：
       - 筛选条件：importance 为 high/medium；actors 与最近 2 场角色有交集 或 category="rule"；scene < current_scene
       - 获取最近 2 场场景中出现的角色：从 `state["scenes"][-2:]` 中提取参与的角色名
       - 返回筛选后的事实列表

    7. **`validate_consistency_logic(state: dict) -> dict`**（per D-12/D-15/D-16）：
       - 调用 _filter_relevant_facts 获取相关事实
       - 无相关事实时返回 `{"status": "success", "message": "✅ 无需检查", "contradictions": [], "facts_checked": 0}`
       - 获取最近 2 场场景内容
       - 返回 `{"status": "success", "relevant_facts": [...], "recent_scenes": [...], "facts_checked": N, "scenes_analyzed": N}`

    8. **`validate_consistency_prompt(facts: list[dict], recent_scenes: list[dict]) -> str`**（per D-14）：
       - 核心指令："对比以下已确立事实与近期场景内容，判断是否存在逻辑矛盾"
       - 矛盾定义："同一时间同一地点不可能同时为真"
       - 非矛盾："时间推移导致的变化、角色视角差异、新信息的补充"
       - 要求："仅报告确信的矛盾，忽略模糊或可解释的差异"
       - 事实列表：编号 + [category] + fact 文本 + 涉及角色
       - 近期场景：场景号 + 内容前 200 字
       - 输出格式：JSON `{"contradictions": [{"fact_id", "fact_text", "scene_text", "explanation"}], "has_contradiction": bool}`

    9. **`generate_repair_narration_prompt(contradiction: dict, repair_type: str) -> str`**（per D-20）：
       - 输入：矛盾描述 + 修复方式（supplement/correction）
       - 补充式指令："用'之前未曾提及的是...'语气补充信息"
       - 修正式指令："用'其实...'、'原来...'语气圆回"
       - 要求："1-2 句自然旁白，不要直接说'这是矛盾'"

    10. **`repair_contradiction_logic(fact_id: str, repair_type: str, repair_note: str, state: dict) -> dict`**（per D-22）：
        - 验证 repair_type 为 "supplement" 或 "correction"
        - 在 established_facts 中查找 fact_id，不存在返回 error
        - **不修改或删除原始事实**——仅追加 `repair_note` 字段：`{"repair_type": ..., "repair_note": ..., "repaired_at": ...}`
        - 更新 state["coherence_checks"]["total_contradictions"]
        - 返回 `{"status": "success", "fact_id": ..., "repair_type": ..., "message": "✅ 矛盾已标记修复"}`

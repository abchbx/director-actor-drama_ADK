2. 改造 `_send_message()` 函数——在 `async for event in runner.run_async(...)` 循环外层包裹 spinner：
   ```python
   async def _send_message(runner: Runner, message: str) -> str:
       content = types.Content(role="user", parts=[types.Part.from_text(text=message)])
       response_parts = []
       _content_keys = [
           "formatted_narration", "formatted_dialogue", "formatted_scene",
           "dialogue", "narration", "message",
       ]
       console = Console()
       spinner = Spinner("dots", text=" 🤔 思考中...")
       spinner_active = False

       try:
           async for event in runner.run_async(
               user_id=USER_ID,
               session_id=SESSION_ID,
               new_message=content,
           ):
               # Start spinner on first event (before final response)
               if not spinner_active and not event.is_final_response():
                   Live(spinner, console=console, transient=True).start()
                   spinner_active = True

               if event.is_final_response():
                   # Stop spinner before printing final response
                   if spinner_active:
                       Live.get().stop()  # or use context manager pattern
                       spinner_active = False
                   if event.content and event.content.parts:
                       for part in event.content.parts:
                           if part.text:
                               response_parts.append(part.text)
               elif event.content and event.content.parts:
                   for part in event.content.parts:
                       if part.function_call:
                           # ... existing function_call display ...
                           # Stop spinner while showing function calls
                           pass
                       if part.function_response:
                           # ... existing function_response display + scene summary ...
                           pass
       except Exception as e:
           if spinner_active:
               # Ensure spinner stopped on error
               pass
           return f"[错误] {e}"
       finally:
           # Ensure spinner is always stopped
           pass

       return "\n".join(response_parts)
   ```

   **重要实现细节：** `rich.live.Live` 是同步上下文管理器，与 async generator 的交互需要特别注意。推荐方案：
   - 使用 `Live` 的 `start()`/`stop()` 方法而非 `with` 语句
   - 在 finally 块中确保 `Live.stop()`
   - 如果 `Live` 与 async 冲突，降级为简单打印 `"⏳ 思考中..."` + 完成后打印 `"✅ 完成"`

3. **降级方案（如果 rich Live 与 async runner 冲突）：**
   - 在 `async for` 循环前打印 `"⏳ 思考中..."`
   - 在收到 `event.is_final_response()` 时打印 `"\r✅ 完成！\n"`
   - 无需 rich 依赖，更简单可靠

2. 新增辅助函数：
   ```python
   def _extract_actors_from_response(resp: dict) -> str:
       """Extract actor names from write_scene response for scene summary."""
       # 从 state["actors"] 的键名中获取，或从 formatted_scene 中解析
       actors = resp.get("actors_in_scene", [])
       if actors:
           return "、".join(actors)
       # Fallback: 从 dialogue_content 中提取 "🎭 角色名" 模式
       dialogue = resp.get("dialogue_content", "") or resp.get("formatted_scene", "")
       import re
       pattern = r"🎭\s*(\S+?)（"
       matches = re.findall(pattern, dialogue)
       if matches:
           return "、".join(matches)
       return ""
   ```

    All 8 endpoints follow this pattern:
    1. Acquire `asyncio.Lock` via `Depends(get_runner_lock)` (STATE-03: serial Runner access)
    2. Check for active drama session (except `/start`) — if `tool_context.state.get("drama", {}).get("theme")` is empty, raise `HTTPException(status_code=404, detail="No active drama session")`
    3. Format message string matching CLI command format
    4. Call `await run_command_and_collect(runner, message, USER_ID, SESSION_ID)`
    5. Return `CommandResponse(**result)`

    Endpoint implementations:

    ```python
    @router.post("/drama/start", response_model=CommandResponse)
    async def start_drama(
        request: StartDramaRequest,
        runner=Depends(get_runner),
        lock=Depends(get_runner_lock),
        tool_context=Depends(get_tool_context),
    ):
        async with lock:
            # D-06: auto-save existing drama before starting new one
            drama_state = tool_context.state.get("drama", {})
            if drama_state.get("theme"):
                from app.state_manager import save_progress, flush_state_sync
                save_progress(save_name="", tool_context=tool_context)
                flush_state_sync()
            result = await run_command_and_collect(runner, f"/start {request.theme}", USER_ID, SESSION_ID)
            return CommandResponse(**result)
    ```

    ```python
    @router.post("/drama/next", response_model=CommandResponse)
    ```

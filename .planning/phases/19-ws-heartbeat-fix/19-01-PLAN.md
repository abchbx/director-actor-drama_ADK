**WebSocketManager.kt onMessage() 重写（核心修复）：**

将当前 onMessage() 中所有 `text.contains("\"type\"") && text.contains("\"xxx\"")` 的字符串匹配替换为结构化 JSON 解析。使用 `json.parseToJsonElement(text).jsonObject` 解析消息，提取 `type` 字段后用 `when` 表达式分发处理：

```kotlin
override fun onMessage(webSocket: WebSocket, text: String) {
    if (thisGeneration != connectGeneration) return

    try {
        val jsonObject = json.parseToJsonElement(text).jsonObject
        val type = jsonObject["type"]?.jsonPrimitive?.content ?: ""

        when (type) {
            "ping" -> {
                // D-01/D-02: 服务端应用层心跳 — 立即回复 pong
                webSocket.send("""{"type":"pong"}""")
                return
            }
            "pong" -> {
                // 服务端对客户端 pong 的回显，或心跳确认 — 忽略
                return
            }
            "replay" -> {
                // D-09/D-10: replay buffer 补发
                val replayMsg = json.decodeFromString<ReplayMessageDto>(text)
                replayMsg.events.forEach { event ->
                    _events.tryEmit(event)
                }
                return
            }
        }

        // 常规事件
        val event = json.decodeFromString<WsEventDto>(text)
        _events.tryEmit(event)
    } catch (e: Exception) {
        Log.e(TAG, "Failed to process WS message: $text", e)
    }
}
```

需要添加 import: `import kotlinx.serialization.json.jsonObject`, `import kotlinx.serialization.json.jsonPrimitive`

**NetworkModule.kt pingInterval 调整：**

将第62行 `pingInterval(30, TimeUnit.SECONDS)` 改为 `pingInterval(60, TimeUnit.SECONDS)`。

原理：OkHttp pingInterval 发送 WS 协议层 Ping 帧（RFC 6455），uvicorn 自动回复 Pong 帧。这不是应用层心跳，而是 TCP 层 keepalive。设为 60s 比服务端应用层心跳 15s 长，避免两层冲突。保留此配置是因为：移动网络 NAT 超时通常 60-120s，WS 协议层 Ping 可以维持 TCP 连接活跃。

- 指数退避重连策略不变

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Server → Client (WS) | 服务端发送的 ping 消息可被客户端信任（同一连接内） |
| Client → Server (WS) | 客户端发送的 pong/heartbeat 消息跨信任边界 — 可能被伪造或注入 |
| Network (WS transport) | WS 连接可能被中间人篡改（无 TLS 时）或 NAT 超时断连 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-01 | Denial of Service | websocket.py receive loop | accept | heartbeat 消息类型已受 MAX_CONNECTIONS=10 限制；单客户端 heartbeat 频率由客户端控制，服务端无法限制接收速率；但 heartbeat 不触发重量级操作，仅 record_pong + send_json，DoS 风险低 |
| T-19-02 | Tampering | WebSocketManager.onMessage JSON parsing | mitigate | 使用 json.parseToJsonElement() 严格解析，非 JSON 消息被 catch 处理，不会触发 pong 响应；比 text.contains() 更安全，避免恶意构造的消息触发误响应 |
| T-19-03 | Spoofing | websocket.py record_pong | accept | 伪造 pong 仅延长超时，不授予额外权限；WS 连接已通过 AUTH-03 token 认证；heartbeat 回复 pong 也不授予权限 |
| T-19-04 | Denial of Service | ConnectionManager.heartbeat loop | mitigate | 15s interval + 30s timeout 限制心跳频率；超时自动断连释放资源；MAX_CONNECTIONS=10 防止连接耗尽 |
</threat_model>

# Phase 22: 群聊模式改造

**Goal:** 将Android剧本详情页从导演指令模式改造为群聊模式：用户输入主题→创建角色→进入群聊界面→@角色对话→角色可主动发言

**Depends on:** Phase 17, Phase 18
**Priority:** HIGH — 核心交互体验升级

## Requirements

- CHAT-01: 用户消息右对齐气泡，角色消息左对齐（现有 DialogueBubble 复用）
- CHAT-02: 输入框改为纯聊天式，删除斜杠命令体系，@角色名 触发提及选择器
- CHAT-03: @提及选择器：输入 `@` 弹出角色列表，选择后插入 `@角色名 `
- CHAT-04: 不 @ 时发群消息（FREE_TEXT → /action），@角色时路由到 /speak
- CHAT-05: 保留 /next /end 为快捷按钮（底部芯片），移除 /action /speak 按钮
- CHAT-06: 后端新增 `POST /drama/chat` 接口，接收 {message, mention?}
- CHAT-07: 角色主动发言：其他角色基于情境可选择性插话（WS 事件 `actor_chime_in`）
- CHAT-08: SceneBubble 增加 UserMessage 类型

## Success Criteria

1. 用户在输入框发消息，显示为右对齐用户气泡
2. 输入 @ 弹出角色选择器，选择后 @角色名 插入输入框
3. @角色 发送消息 → 路由到 /speak，角色回复显示为左对齐气泡
4. 不 @ 发送消息 → 路由到 /action（群消息）
5. /next /end 保留为底部快捷按钮
6. 角色可主动发言（WS 推送 actor_chime_in 事件）
7. 体验为"和角色们群聊"，而非"导演下指令"

## Critical Gaps

### Gap 1: WS Heartbeat Pong Missing (Severity: CRITICAL)

**Server side** (Phase 14): `websocket.py` sends `{"type": "ping"}` every 15s, expects `{"type": "pong"}` within 30s.

**Android side** (Phase 18): `WebSocketManager.onMessage()` tries to parse ping as `WsEventDto` and silently fails. Never sends `{"type": "pong"}` back.

**Impact**: Every WS connection drops after 30s. Auto-reconnect kicks in, creating a connect→30s→disconnect→reconnect thrash cycle. Real-time scene push is unreliable.

**Fix**: Add pong handling in `WebSocketManager.onMessage()`:
```kotlin
if (text == """{"type":"ping"}""") {
    webSocket.send("""{"type":"pong"}""")
    return
}
```

**Affected requirements**: WS-05, APP-04, APP-15

### Gap 2: isProcessing Never Resets (Severity: CRITICAL)

**Location**: `DramaDetailViewModel.sendCommand()`

**Issue**: `isProcessing = true` set when sending command. On success, no code sets `isProcessing = false`. Only the failure handler resets it. `CommandInputBar` uses `enabled = !isProcessing`.

**Impact**: After any successful command, the command input bar is permanently disabled. User cannot send a second command.

**Fix**: Add `isProcessing = false` in the success path of `sendCommand()`, or reset it when WS typing/content events arrive.

**Affected requirements**: APP-05

## High-Priority Gaps

### Gap 3: 4 API Endpoints Unwired (Severity: HIGH)

Backend endpoints `/steer`, `/auto`, `/storm`, `/export` are defined in `DramaApiService` but have no corresponding methods in `DramaRepository`. The Android app cannot invoke these endpoints.

- `/steer` → routes to FREE_TEXT (userAction) instead of steer API
- `/auto` → routes to FREE_TEXT (userAction) instead of auto API
- `/storm` → not accessible from Android
- `/export` → APP-09 broken (no Repository method, no ViewModel logic, no UI)

**Affected requirements**: API-02 (partial), APP-05 (partial), APP-09 (broken)
